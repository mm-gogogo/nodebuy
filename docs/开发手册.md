# NodeBuy 开发手册

面向需要二次开发本项目的工程师。

## 一、技术栈

| 层 | 技术 | 版本 |
| --- | --- | --- |
| CMS / 后台 | Payload CMS | 3.85.1 |
| 前端框架 | Next.js（App Router） | 16.2.6 |
| UI | React | 19.2.6 |
| 数据库 | PostgreSQL（`@payloadcms/db-postgres`，底层 Drizzle） | 16 |
| 富文本 | Lexical（`@payloadcms/richtext-lexical`） | — |
| 语言 / 包管理 | TypeScript 5.7 / pnpm 10 | — |

前台与后台跑在**同一个 Next.js 进程**里：`src/app/(frontend)` 是内容站，`src/app/(payload)` 是 Payload 管理面板与 REST/GraphQL API，无需分开部署。

## 二、本地环境搭建

```bash
# 1. 数据库（Docker，宿主端口 5434）
docker compose up -d

# 2. 依赖
pnpm install

# 3. 环境变量
cp .env.example .env
#    DATABASE_URL      Postgres 连接串
#    PAYLOAD_SECRET    JWT 签名密钥，生产环境务必换成强随机值
#    NEXT_PUBLIC_SERVER_URL  站点对外地址

# 4. 演示数据（可选）
npx tsx src/seed.ts

# 5. 启动
pnpm dev
```

开发模式下 Payload 会自动把集合结构 push 到数据库（Drizzle dev push），改完集合定义刷新即可，无需手写迁移。

> **注意**：种子脚本必须用 `npx tsx src/seed.ts` 运行。`payload run src/seed.ts` 在本项目会静默失败（无法解析 `@payload-config` 路径别名）。脚本有幂等保护——已有数据时直接跳过。

## 三、目录结构

```
src/
├── payload.config.ts          # Payload 总配置（集合注册、数据库适配器）
├── payload-types.ts           # 自动生成的类型，不要手改
├── seed.ts                    # 演示数据种子脚本
├── collections/               # 集合定义（数据模型）
│   ├── Providers.ts           #   服务商
│   ├── Plans.ts               #   套餐
│   ├── Reviews.ts             #   测评（带草稿版本）
│   ├── Rankings.ts            #   榜单
│   ├── Deals.ts               #   优惠
│   ├── Users.ts / Media.ts    #   管理员 / 媒体库
├── components/
│   ├── ui.tsx                 # 服务端共享组件（徽标、评分条、AFF 按钮、栏目头）
│   └── CopyCode.tsx           # 客户端组件：优惠码点击复制（含成功/失败态）
├── lib/
│   └── labels.ts              # 线路/分类/运营商的中文映射 + 规格/价格格式化
└── app/
    ├── (frontend)/            # 前台
    │   ├── tokens.css         #   设计令牌（颜色/字体/间距，全站唯一取色来源）
    │   ├── styles.css         #   全站样式（只允许引用 tokens 里的变量）
    │   ├── layout.tsx         #   导航 + 页脚 + 字体加载
    │   ├── page.tsx           #   首页（五个发现面）
    │   ├── rankings/ reviews/ providers/ deals/   # 各列表与详情页
    ├── (payload)/             # Payload 后台与 API（脚手架生成，一般不动）
    └── go/[slug]/route.ts     # AFF 302 跳转路由
```

## 四、数据模型速览

| 集合 | 关键字段 | 说明 |
| --- | --- | --- |
| `providers` | `slug`(唯一)、`affUrl`、`scores{}`、`overallScore`、`datacenters[]`、`pros/cons[]`、`brandColor` | `brandColor` 用于无 Logo 时的字母徽标底色 |
| `plans` | `provider`(关联)、`cpuCores/ramMB/storageGB`、`trafficTB`(0=不限)、`priceMonthly/priceYearly`、`route`(枚举)、`affUrl`、`inStock` | `route` 枚举：cn2gia / cn2gt / cmin2 / 9929 / 4837 / direct / bgp |
| `reviews` | `content`(Lexical)、`scores{}`、`benchmarks{gb5、磁盘、speedtests[]、routes[]}`、`_status` | 开了 `versions.drafts`，支持草稿/发布流 |
| `rankings` | `category`(6 个枚举)、`featured`、`items[]`(数组顺序即名次) | 首页精选榜取 `featured=true` 的第一个 |
| `deals` | `code`、`discount`、`expiresAt`、`featured` | 前台按 `-featured` 排序 |

修改集合后执行 `pnpm generate:types` 重新生成 `payload-types.ts`。

## 五、前台数据流

页面全部是 **React Server Components**，直接走 Payload Local API（不发 HTTP）：

```ts
const payload = await getPayload({ config })
const { docs } = await payload.find({ collection: 'rankings', where: { slug: { equals: slug } } })
```

每个页面导出 `export const revalidate = 60`（ISR）：后台改内容最多 60 秒后前台生效。需要即时生效可改小或在写钩子里调 `revalidatePath`。

关联字段默认 `depth=2` 已展开成对象，模板里用 `typeof item.provider === 'object'` 防御未展开的情况。

## 六、AFF 跳转机制

`src/app/go/[slug]/route.ts`：

- `/go/<provider-slug>` → 302 到服务商 `affUrl`，没填则官网，再没有则回服务商详情页
- `/go/<provider-slug>?plan=<id>` → 优先用套餐级 `affUrl`（校验套餐确属该服务商）
- 前台所有出站按钮统一走 `AffButton` 组件，带 `rel="nofollow noopener sponsored"`

要加点击统计：在 route.ts 的 302 之前写一条记录（可新建 `clicks` 集合或打到日志/分析平台）。

## 七、设计系统约定（改样式前必读）

- 颜色、字体、间距**只能引用 `tokens.css` 里的变量**，禁止在 styles.css 里写裸 hex/oklch；需要新值先在 tokens 里命名再引用。
- 间距走 4pt 标尺（`--space-*`）；数据数字一律 `font-variant-numeric: tabular-nums` + `--font-mono`。
- 字体上限三族：Geist（标题正文）、Noto Sans SC（中文回落）、Geist Mono（字标与数据）。
- 交互元素至少具备 default / hover / focus-visible / active / disabled 五态；焦点环即时出现、不做过渡。
- `html, body { overflow-x: clip }` 不要移除——这是移动端横向滚动的安全网。
- 文件头部的 `/* Hallmark · ... */` 注释是设计系统戳，重设计时保留并更新。

## 八、常用命令

```bash
pnpm dev                  # 开发（自动 push 数据库结构）
pnpm build && pnpm start  # 生产构建与启动
pnpm generate:types       # 集合定义 → TS 类型
pnpm generate:importmap   # 给后台加自定义组件后执行
npx tsc --noEmit          # 类型检查
pnpm lint                 # ESLint
npx tsx src/seed.ts       # 种子数据
```

## 九、常见开发任务

**给套餐加一个字段**（例：IPv6 数量）：
1. `src/collections/Plans.ts` 的 `fields` 里加 `{ name: 'ipv6Count', label: 'IPv6 数量', type: 'number' }`
2. `pnpm generate:types`
3. 前台需要展示的话改 `lib/labels.ts` 的 `specLine()` 或对应页面

**加一个榜单分类**：`Rankings.ts` 的 `category.options` 加一项，同时在 `lib/labels.ts` 的 `categoryLabels/categoryDescriptions` 补中文。

**换数据库**：`.env` 改 `DATABASE_URL` 后重启即可，开发模式自动建表；生产环境见部署手册的迁移章节。
