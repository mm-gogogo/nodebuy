# NodeBuy 使用手册

面向日常运营内容的站长，不需要懂代码。

## 一、登录后台

浏览器打开 `https://你的域名/admin`。

演示数据自带管理员：`admin@nodebuy.local` / `nodebuy123`（正式运营前请在右上角头像 → Account 里改密码）。

后台左侧「内容」分组下共五个集合：**服务商、套餐、测评、榜单、优惠速递**。前台页面每 60 秒自动刷新一次缓存，保存后稍等片刻即可在前台看到变化。

## 二、核心概念：AFF 链接怎么生效

这是本站作为推广站的核心机制，先弄明白：

1. 每个**服务商**有一个「AFF 推广链接」字段——把你在该商家后台拿到的推广链接填进去（如 `https://bandwagonhost.com/aff.php?aff=12345`）
2. 每个**套餐**也有可选的「AFF 链接」——填了就优先用它（适合指向具体套餐的下单页）
3. 前台所有「入手 / 去下单 / 官网直达」按钮都指向 `你的域名/go/服务商slug`，访客点击后 302 跳到上面填的链接

**优先级：套餐 AFF > 服务商 AFF > 服务商官网。**

所以换推广 ID 只需要在后台改一个字段，全站按钮立即生效，不用动任何页面。

> 种子数据里所有 AFF 链接都是 `aff=0000` 占位符，**正式运营前务必逐个换成你自己的**。

## 三、日常运营流程

### 收录一家新服务商

1. 「服务商」→ Create New
2. 必填：名称、Slug（小写英文短横线，决定网址，**建好后别再改**）
3. 建议填全：一句话定位、简介、官网、**AFF 链接**、品牌色（无 Logo 时字母徽标的底色，填 hex 如 `#2b6cb0`）、成立年份、付款方式、数据中心（勾「大陆优化线路」的机房会在前台高亮）、四项评分、综合评分、优缺点
4. 保存后前台「收录服务商」栏目自动出现

### 添加套餐

「套餐」→ Create New，选所属服务商，填规格（核数/内存 MB/硬盘 GB/月流量 TB，流量填 0 表示不限）、价格（月付年付至少填一个，单位美元）、机房、线路。

- **线路**字段（CN2 GIA / CMIN2 / 9929 等）会在前台显示成蓝色徽标，是访客最关心的信息，别空着
- 商家缺货时把「有货」取消勾选，前台按钮自动变成「缺货」灰字，不用删数据

### 发一篇测评

1. 「测评」→ Create New，填标题、Slug、关联的服务商与套餐
2. **摘要**：两三句话，显示在列表页和首页
3. **正文**：富文本编辑器，支持标题/列表/链接/图片
4. **分项评分**：性能/网络/性价比/售后（0–10，可一位小数）
5. **跑分数据**：CPU 型号、Geekbench 5 单核多核、磁盘读写；「测速节点」每行一个城市运营商；「回程线路」每行一个运营商的线路说明——这些会在前台渲染成规整的表格
6. **结论**：一段话，前台显示为左侧竖线的引用块
7. 右上角 **Publish changes** 发布（Save Draft 是存草稿，前台不显示）

### 维护榜单

「榜单」是前台的门面，建议每月更新一次：

- **上榜条目的先后顺序就是名次**，拖动条目左侧的把手排序
- 每个条目选服务商 + 代表套餐，填评分、上榜理由、适合人群
- 勾选「**首页精选**」的榜单显示在首页第一屏（只取一个，建议只给综合榜勾）
- 「评选标准」写清权重（如：网络 40% + 性能 25%…），显示在榜单底部，提升可信度
- 建议把标题带上月份（如「2026 年 7 月 · VPS 综合推荐榜」），更新时直接改标题与条目

### 发布优惠

「优惠速递」→ Create New：标题、服务商、优惠码（前台点击自动复制）、折扣描述、失效日期。

- 勾「首页置顶」的优惠排在前面
- **过期的优惠请及时删除或下架**——前台原样展示列表，挂着失效码很伤信任

## 四、内容质量建议

- 测评数字要可信：跑分、测速、延迟互相要对得上（万兆口跑出 100Mbps 这种自相矛盾会被读者识破）
- 结论别全是夸：优缺点都写，缺点写真的，转化率反而更高
- 标题写信息量：「搬瓦工 DC6 实测：三网 GIA 回程，晚高峰依旧稳」好于「搬瓦工评测」
- 页脚已自带「部分外链为推广链接（AFF）」声明，这是行业惯例也是合规要求，不建议删

## 五、常见问题

**改了内容前台没变？** 等 60 秒（缓存周期）。还不行就刷新页面强制重载（Ctrl/Cmd+Shift+R）。

**Slug 能改吗？** 技术上能，但改了等于换网址，已被搜索引擎收录/别人转发的旧链接会 404。建好就别动。

**删服务商会怎样？** 它名下的套餐、测评、榜单条目会留下空引用，前台对应条目自动隐藏。正确做法是先清理关联内容再删，或干脆留着不删只下架套餐。

**想换站名/页脚？** 站名与页脚文案在源码 `src/app/(frontend)/layout.tsx` 里，让开发同学改（见开发手册）。

**忘记管理员密码？** 登录页点 Forgot password 走邮件重置（需配邮件服务），或让开发同学用种子脚本逻辑在服务器上重建账号。

## 六、账号安全

- 管理员账号建议一人一号：「Users」集合里可以新建
- 后台地址 `/admin` 不要外传；有条件的在反代/Cloudflare 上限制访问 IP
- 定期备份数据库（见部署手册第七节）——内容是这个站最值钱的资产
