# NodeBuy 部署手册

从零把站点部署到一台 Linux 服务器（以 Ubuntu 22.04/24.04 为例），并给出 Vercel 托管方案。

## 〇、前置要求

- Node.js ≥ 20.9（推荐 22 LTS）、pnpm ≥ 9
- PostgreSQL ≥ 14（本手册用 Docker 跑 16）
- 一个解析到服务器的域名（做 AFF 推广域名就是它）

## 一、准备代码与环境变量

```bash
# 解压源码包（或 git clone）
cd /opt && unzip nodebuy-*.zip && cd nodebuy

pnpm install

cp .env.example .env
vim .env
```

生产环境 `.env` 必须改这三项：

```ini
# 数据库连接（按实际部署修改）
DATABASE_URL=postgresql://nodebuy:<强密码>@127.0.0.1:5434/nodebuy

# 随机密钥，泄露等于后台沦陷。生成：openssl rand -hex 32
PAYLOAD_SECRET=<openssl rand -hex 32 的输出>

# 站点对外地址（带 https）
NEXT_PUBLIC_SERVER_URL=https://你的域名.com
```

## 二、数据库

```bash
# 用项目自带的 docker-compose（含数据卷持久化）
docker compose up -d

# 确认
docker exec nodebuy-postgres pg_isready -U nodebuy
```

> 改密码：编辑 `docker-compose.yml` 的 `POSTGRES_PASSWORD` 并同步改 `DATABASE_URL`。
> 已有现成 Postgres 的话跳过本节，直接把 `DATABASE_URL` 指过去。

### 建表（生产环境二选一）

生产模式（`NODE_ENV=production`）下 Payload **不会**自动建表，首次部署需要初始化结构：

**方式 A · 迁移文件（推荐，正规流程）**

```bash
npx payload migrate:create   # 由当前集合定义生成迁移文件（migrations/）
npx payload migrate          # 应用到数据库
```

以后每次改了集合结构，重复以上两步并把 `migrations/` 提交进版本库。

**方式 B · 开发模式 push（快但只适合首次/演示）**

```bash
pnpm dev   # 启动一次让 Drizzle 自动建表，Ctrl+C 退出
```

### 灌演示数据（可选）

```bash
npx tsx src/seed.ts
# 管理员: admin@nodebuy.local / nodebuy123 —— 上线后立刻改密码或删号重建
```

## 三、构建与启动

```bash
pnpm build
pnpm start          # 默认 3000 端口；改端口：pnpm start -p 3001
```

### 用 PM2 守护（推荐）

```bash
npm i -g pm2
pm2 start "pnpm start" --name nodebuy --cwd /opt/nodebuy
pm2 save && pm2 startup     # 开机自启
pm2 logs nodebuy            # 看日志
```

### 或 systemd

```ini
# /etc/systemd/system/nodebuy.service
[Unit]
Description=NodeBuy
After=network.target docker.service

[Service]
WorkingDirectory=/opt/nodebuy
ExecStart=/usr/bin/env pnpm start
Restart=always
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload && systemctl enable --now nodebuy
```

## 四、Nginx 反代 + HTTPS

```nginx
# /etc/nginx/sites-available/nodebuy
server {
    listen 80;
    server_name 你的域名.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 媒体库上传文件（后台 Logo 等）由 Next 直接服务，无需额外配置
    client_max_body_size 20m;   # 放开上传大小
}
```

```bash
ln -s /etc/nginx/sites-available/nodebuy /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx

# HTTPS（Let's Encrypt）
apt install certbot python3-certbot-nginx
certbot --nginx -d 你的域名.com
```

套 Cloudflare 的话：DNS 开橙云，SSL 模式选 Full (strict)，`/admin*` 建议加 WAF 规则限制访问来源。

## 五、Vercel 托管方案（免运维）

1. 代码推到 GitHub，Vercel 导入仓库（自动识别 Next.js）
2. 数据库用 Neon / Supabase / Vercel Postgres，把连接串填到环境变量 `DATABASE_URL`（注意加 `?sslmode=require`）
3. 环境变量同样配 `PAYLOAD_SECRET`、`NEXT_PUBLIC_SERVER_URL`
4. 首次部署前用迁移方式建表（本地 `npx payload migrate` 指向云库），种子同理
5. 媒体上传在 Vercel 无持久磁盘，需装 `@payloadcms/storage-s3`（或 vercel-blob 适配器）把 Media 集合指向对象存储

## 六、升级与回滚

```bash
cd /opt/nodebuy
git pull            # 或解压新包覆盖（保留 .env 与 media/）
pnpm install
npx payload migrate # 如有新迁移
pnpm build
pm2 restart nodebuy
```

回滚：保留上一版目录（如 `/opt/nodebuy-prev`），出问题切回去重启即可；数据库迁移回滚用 `npx payload migrate:down`。

## 七、备份

```bash
# 数据库（建议 cron 每日）
docker exec nodebuy-postgres pg_dump -U nodebuy nodebuy | gzip > /backup/nodebuy-$(date +%F).sql.gz

# 媒体文件
tar czf /backup/media-$(date +%F).tar.gz /opt/nodebuy/media

# 恢复
gunzip -c nodebuy-2026-06-10.sql.gz | docker exec -i nodebuy-postgres psql -U nodebuy nodebuy
```

## 八、上线检查清单

- [ ] `PAYLOAD_SECRET` 已换强随机值
- [ ] 数据库密码已改、5434 端口未对公网开放（`docker-compose.yml` 里可改成 `127.0.0.1:5434:5432`）
- [ ] 种子管理员密码已修改
- [ ] `NEXT_PUBLIC_SERVER_URL` 是正式域名
- [ ] 所有服务商/套餐的 `affUrl` 已换成你自己的推广 ID（种子里是 `aff=0000` 占位）
- [ ] HTTPS 生效、`/go/xxx` 302 正常
- [ ] 备份 cron 已配置
